Respiration Rate Algorithm



In this project, I analyzed breathing signals from a non-contact vitals system monitor to extract breathing rates.

Repository with code can be found here.

Background

The following project was a candidate assessment where I was given several data-sets consisting of breathing signals and was asked to generate an algorithm to determine the overall respiration rate. Due to IP concerns, I cannot openly share the data. However, I would like to present the methodology as well as my code for generating a solution.

The data provided was infant breathing signals from a non-contact vitals montioring system using Doppler-based radio-frequency waves. Several papers have suggested a similar kind of system, which I will leave linked here [Hall et. al, Matthews et. al].

Data Visualization

Using the provided sampling frequency of the signal, the time scale of the signal was reconstructed and one minute of data was extracted and plotted.

It is clear this signal requires some amount of processing to reduce noise and smooth the identified breathing peaks. As such, the FFT of the signal was computed and plotted in the next step.

Pre-processing - Spectral Analysis

The one-sided DFT of the signal was computed and plotted as shown on the left. The frequency axis was normalized as MATLAB uses normalized frequency as parameters for its filter functions.

Analysis of this spectral plot does require some domain knowledge to understand what we're looking at. It is known that the frequency range of respiratory signals is relatively low at approximately 0.1-0.5 hz (source). The resolution of the graph on the left is not great, but it is likely this is the dominant frequency component. Thus, I designed a low-pass filter at a normalized cut-off frequency of 0.1 pi rads/sample to filter out any higher frequency components that may be contaminating the signal.

Pre-processing - Filtering

I used a 10th order Butterworth filter at a 0.1 normalized cutoff frequency to reduce noise. The Butterworth filter is the most ideal filter to use in this case as it has a relatively low amount of attenuation in the passband as opposed to other traditional linear filters (more on that here).

The plot of the right shows the raw respiration signal on the top and the filtered respiration signal on the bottom; significantly better!

Algorithm Development

I employed two different algorithms to calculate the respiration rate of the filtered signal:

1. A simple peak detection algorithm.
2. A moving average filter followed by the same peak detection algorithm.

Pros and cons of each method are discussed below. These algorithms are not intended to be production-ready or particularly robust, but simply a starting point towards refinement of a more advanced algorithm following several trial-validation cycles. The figure below shows the output of both algorithms on a test data-set. Specific accuracy of both algorithms could not be verified as true breathing rate figures were not provided. However, these rates were accurate based on review with the team.

Algorithm 1 - Simple Peak Detection



The peak detection algorithm used a relatively simple method to detect peaks in the waveform. The algorithm traverses each point in the given dataset and compares the previous data point and the next data point to the current. If the current data point is greater than both neighboring data points, the current data point is identified as a peak.

After traversing the entire data-set, the total peak count is then divided by the time frame to calculate the overall respiration rate.

Considerations:

- Not particularly robust. Will detect small peaks and multiple peaks within a single peak. However, this may be actually a positive considering infantile breathing is very rapid and can be very weak.

- Efficiency can possibly be improved, especially in low power embedded settings. The current algorithm will work in linear time for N inputs.

- A baseline can be used as a threshold of comparsion (such as the mean of the signal) to ignore small detected peaks. Again, this must be concerned in relation to the application.

Algorithm 2 - Moving Average Filter



The moving average filter uses the same methodology as algorithm 1 with one key difference; prior to peak determination, a moving average filter is applied to the data-set to smooth transients.

The moving average filter helps with removing noisy/transient data, such as multiple peaks occuring with a larger peak.


Considerations:

- A detriment when lower quality/weaker signals are recorded.

- Efficiency is worse off, since the algorithm must traverse the entire vector first to calculate the moving average and then again to calculate the peaks. This will work in exponential time (N^2 for N inputs)
